## Project Overview

Frontier Corm is an AI-powered autonomous NPC system for [Eve Frontier](https://evefrontier.com) on Sui. It's a monorepo with five services: Sui Move smart contracts, a Go game server (continuity-engine), a TypeScript event indexer, a React web app, and AWS CDK infrastructure.

**Caution:** Eve Frontier is migrating from EVM to SUI. External resources may reference EVM patterns — ignore those.

## Common Commands

```bash
# Local development (full stack)
cp .env.example .env
mprocs                              # starts all services in dependency order

# Testing
make test                           # all tests (Go + Move)
make test-go                        # continuity-engine Go tests
make test-contracts                 # Sui Move contract tests (all 6 packages)
cd continuity-engine && go test ./internal/puzzle/...  # single Go package

# Building
make build                          # all TypeScript projects
make zk-build                       # Groth16 ZK circuit artifacts

# Local docker subset (indexer + postgres + continuity-engine)
make local
make local-down                     # stop, keep data
make local-reset                    # stop + delete volumes

# Deployment (Makefile-driven, targets AWS)
make deploy-env ENV=utopia          # deploy everything
make deploy-images ENV=utopia       # docker build + ECR push
make deploy-frontend ENV=utopia     # vite build + S3 sync + CloudFront invalidation
make deploy-indexer ENV=utopia      # build + push + redeploy indexer only
make deploy-continuity ENV=utopia   # build + push + redeploy continuity-engine only
make publish-contracts ENV=utopia   # publish Move contracts
make upgrade-contracts ENV=utopia   # upgrade Move contracts (preferred over publish)
```

## Architecture

```
Browser (React :5173) --REST--> Indexer (TS :3100) <--events-- Sui Contracts (Move)
       |                              |
       | iframe                       | Postgres (pgvector)
       v                              |
Continuity Engine (Go :3300) ---------+-----> Sui RPC
```

**Contracts** (6 Sui Move packages): `corm_auth` (witness registry), `corm_state` (shared state), `tribe` (membership), `trustless_contracts` (coin/item escrow exchanges), `witnessed_contracts` (build request bounties with Ed25519 verification), `assembly_metadata` (deprecated).

**Continuity Engine** (Go): Three-phase game server (awakening -> puzzles -> contracts). HTMX partial templates. Deterministic trait evolution and reasoning. Two goroutines: HTTP server + event processor communicating via channels. In-memory session store requires ALB sticky sessions.

**Indexer** (TypeScript): Subscribes to Sui checkpoint events, archives to Postgres with checkpoint proofs. REST API for historical queries. Optional cleanup worker (expires stale contracts), witness service (signs build attestations), and Shadow Location Network (encrypted POD storage + ZK proof verification).

**Web** (React): SPA with Sui wallet connection via dapp-kit. Identity resolver maps wallet to Eve Frontier Character. Pages: Dashboard, Tribes, Contracts, Continuity Engine (iframe), Events, Locations, Forge Planner, Settings.

**Infra** (AWS CDK): ECS Fargate, RDS Postgres 16, S3 + CloudFront, ALB with sticky sessions, Route 53. Two environments: utopia (testnet A) and stillness (production testnet).

## Workflow

- **Feature artifact root:** `./plan/artifacts` — phase artifacts (DISCOVERY.md, SPEC.md, etc.) are written to `./plan/artifacts/<feature-slug>/`

## Key Conventions

- **Commit messages:** `<type>(<scope>): <description>` — types: feat, fix, refactor, chore, docs; scopes: web, indexer, continuity-engine, contracts, infra, chain, scripts
- **Design docs:** Each service has a `design-doc.md` at its root. Plans should always reference and update relevant design docs.
- **Contract changes:** Strongly prefer upgrades over full publishes. Notify the user when a publish is absolutely required.
- **Plan naming:** Preface plan names with `[frontier-corm][service-name]`
- **Environment files:** `.env.localnet` is auto-generated by publish scripts. `.env.utopia` and `.env.stillness` are deployment configs. Never commit actual `.env` files (only `.example` templates).

## Services With Tests

- `continuity-engine/` — Go `testing` (ciphers, puzzles, handlers, contracts, traits)
- `contracts/` — Sui Move `#[test]` in all 6 packages
- `dev-tools/transfer_exploit_poc/` — Move security tests

Services **without** tests: indexer, infra, static-data, training-data.

The `web` package has vitest (`vitest run`) with colocated `*.test.ts` / `*.test.tsx` files. jsdom + `@testing-library/react` and Playwright (`npm run test:e2e`) are being introduced with the map-feature — after that, both component and E2E tests are viable.
